Use the route closest to the work.

Ordivon currently uses public GitHub surfaces instead of a contact form or domain mailbox. This keeps project context, change history, and discussion attached to the relevant repository.

Security

Do not publish active secrets or exploit details.

Never place passwords, API keys, personal financial data, authentication tokens, recovery codes, or active exploit details in a public issue. For a sensitive concern, identify the affected repository through the GitHub profile and request a private reporting path without including the secret or exploit itself. Follow any repository-specific `SECURITY.md` guidance where present.

Unavailable routes

No form, mailbox, or support desk yet.

There is currently no `@ordivon.com` public mailbox, customer-support system, account portal, live chat, newsletter, or service-level commitment. The site does not collect contact details.
