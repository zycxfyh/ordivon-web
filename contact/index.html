Useful context

Make a correction or proposal easy to inspect.

Identify the relevant page, file, workflow, claim, or question. Separate the expected result from the observed result, link supporting evidence where possible, and state whether the issue concerns current fact, interpretation, or presentation.

Security

Keep sensitive material out of public issues.

Do not publish passwords, API keys, authentication tokens, recovery codes, personal financial data, or active exploit details. Identify the affected repository and follow its `SECURITY.md` guidance or request a private reporting route.
