Runtime · Core accepted · Production adoption

Durable execution for capable agents.

Ordivon Runtime is a local execution and recovery layer. The agent owns reasoning and operation; the runtime owns durable execution truth, process identity, bounded observation, cancellation, artifacts, and recovery after interruption.

Purpose

Keep execution reliable without turning the agent into a captive workflow.

Agent reasoning can be flexible, exploratory, and model-specific. Host execution cannot be ephemeral. Commands need stable identity, observable state, bounded output, cancellation, restart recovery, and durable artifacts even when the conversation or client connection disappears.

Ordivon isolates that small non-reconstructable core. Git, source, tests, and live system state remain the primary truth; the runtime persists only the execution facts required to continue or recover work.

Execution path

A thin runtime between intent and the host.

01Human intent

Defines the objective, constraints, and delegated authority.

02Agent operation

Reasons, chooses tools, and adapts implementation to observed results.

03Durable execution

Owns Job and Attempt identity, process lifecycle, output bounds, and artifacts.

04Recovery or continuation

Lets a later client observe, cancel, inspect, or continue from durable facts.

Engineering defaults

Thin context, thin recovery, explicit responsibility.

Thin context

Use source, tests, Git, live state, and a few current documents instead of reconstructing the repository as prose.

Thin recovery

Persist only execution facts that cannot be reconstructed from ordinary engineering truth.

Reallocated responsibility

The user owns intent, the agent owns implementation and operation, and the runtime owns durable execution truth.

Growth by failure

Add a capability only when repeated real tasks prove that ordinary host command execution cannot provide the required semantics.
