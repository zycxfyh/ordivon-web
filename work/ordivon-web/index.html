Ordivon Web is the public entrypoint for the portfolio. It gives projects, research, notes, current priorities, and contribution routes a stable address while leaving implementation truth in each canonical repository.

Role

Connect public understanding without duplicating repository truth.

The website explains what each project is, how the portfolio fits together, what is being worked on now, and where public evidence lives. It does not become a second registry of mutable technical facts.

Project implementation, detailed architecture, tests, issues, pull requests, and current capability claims remain in their source repositories. The site provides orientation and authored interpretation.

Release contract

Keep the site deployable, inspectable, and easy to recover.

01Dependency-free source

No framework, package manager, backend, external font, remote script, or image dependency.

02Automated verification

Check required routes, metadata, canonical URLs, links, assets, sitemap, manifest, CNAME, and selected tracking patterns.

03Public release history

Use Git commits, pull requests, CI, and GitHub Pages deployment as the recoverable record.

04DNS boundary

Normal content releases do not modify the separate apex-domain or GoDaddy configuration.
