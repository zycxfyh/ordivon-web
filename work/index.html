Relationship

The projects exchange methods and lessons without sharing one ontology.

Ordivon Runtime can support engineering work without becoming a FinHarness component. FinHarness can adopt useful execution and recovery patterns without turning its financial domain into general infrastructure. Ordivon Web can explain both without becoming the canonical registry of either.

Source code, tests, data, Git identity, and each repository’s current-state material remain the owners of current facts. The website provides orientation, interpretation, and routes to evidence.
